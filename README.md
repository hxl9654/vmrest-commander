<h2 id="english">🇬🇧 English</h2>

**VMRest-Commander** is a lightweight, centralized web dashboard and automation toolset for managing VMware Workstation instances across a local network via the native vmrest API. By enabling and configuring VMware's official `vmrest` interface and combining it with an intuitive Web UI, it provides cross-host VM status monitoring and power management (Power On, Power Off, Suspend, etc.).

### 🌟 Features

- **Automated Configuration (Init Server)**: Provides an automated PowerShell script to set up `vmrest` credentials, configure firewall rules, set up port forwarding (`netsh portproxy`), and create a scheduled task for silent background auto-start. Now includes configuration preservation, `VMAuthdService` recovery, hourly task repetition, and debug logging (`vmrest.log`) for enhanced stability.
- **Batch Host Scanning (Web Management)**: A lightweight Node.js management backend that supports scanning a custom IP range for online hosts.
- **Global VM Overview**: Displays all VMs across all hosts as cards in a unified Web UI, showing real-time power states (Powered On, Powered Off, Suspended, etc.).
- **Power Management**: Control your VMs directly from the web interface: Power On, Power Off, Shutdown, Suspend, Resume, Pause, and Unpause. Supports **Batch Start** with a phased progress bar.
- **Hide Specific VMs**: Support hiding specific VMs in the UI (e.g., template machines or test environments) that don't require attention. Skips polling for hidden VMs to improve performance.

### 🚀 Architecture

This project consists of two independent but cooperative parts:

1. **`initserver` (Host Machine Side)**:
   Runs on **each** host machine where VMware Workstation Pro is installed. It is responsible for exposing the VMware REST API for that machine.
2. **`manage` (Management Control Side)**:
   Runs on **any** machine in the LAN (can be one of the hosts or a standalone server). It provides the Web interface, scans, and centrally dispatches commands to all hosts.

### 📦 Deployment & Usage Guide

#### Step 1: Host Setup

Perform the following steps on **every** Windows host running VMware Workstation that you want to manage:

1. Go to the `initserver` directory:
   Open PowerShell as **Administrator** and navigate to the `initserver` directory.
2. Run the initialization script:
   ```powershell
   .\init.ps1
   ```
   *(If you encounter execution policy restrictions, run `Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process` first)*
3. Follow the on-screen prompts:
   - The script will automatically locate `vmrest.exe` and prompt you to set a **REST API username and password** (it is recommended to use the same credentials on all hosts for easier management).
   - Select the LAN IP address to listen on (it automatically detects your internal IP).
   - The script will configure the firewall, set up port forwarding, and create a Windows scheduled task named "VMware REST API Background Service".
   - **Note:** The script supports the `-UsePasswordLogon` parameter. Using this parameter allows the background task to start silently even when no user is logged in (requires entering your Windows account password).

#### Step 2: Management Server Setup

Choose a machine to run the Web Management Dashboard:

1. Navigate to the `manage` directory:
   ```bash
   cd manage
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Configure settings:
   Copy `config-sample.json` and rename it to `config.json` (if you start directly, it will be automatically copied). Open `config.json` and edit:
   ```json
   {
     "startIp": "192.168.1.100",  // Starting IP of the hosts to scan
     "endIp": "192.168.1.110",    // Ending IP of the hosts to scan
     "username": "YOUR_USERNAME", // vmrest username set in Step 1
     "password": "YOUR_PASSWORD", // vmrest password set in Step 1
     "logLevel": "warn",          // Log output level (debug, info, warn, error)
     "hiddenVMs": [],
     "bootSequence": [            // Optional: Defines the order for Batch Start by matching VM names with wildcards
       "k8s-etcd-*",
       "db-*",
       "k8s-lb-*",
       "k8s-ctl-*",
       "k8s-wrk-*"
     ]
   }
   ```
4. Start the service:
   ```bash
   npm start
   ```
   *(Alternatively, you can just double-click `run.bat` in the `manage` directory on Windows to start the server and automatically open the UI in your browser.)*
   *(For long-term running, it's recommended to use `pm2` or register it as a system service: `pm2 start server.js --name "vmrest-manage"`)*

#### Step 3: Access the Web UI

Open your browser and navigate to the IP of the management server on port 3001, for example:
`http://localhost:3001` or `http://192.168.1.x:3001` *(If you used `run.bat`, it will open automatically).*

- Click the **Scan Network** button to scan the configured IP range.
- Once the scan completes, you can view the VM status of all online hosts and control their power states.
- **Auto Scan** is enabled by default with a 30-second interval for real-time monitoring.

### ⚠️ Security & Important Notes

- **Credential Security**: Do not commit the `manage/config.json` file containing real credentials to a public Git repository (this file is already ignored in `.gitignore`).
- **Network Isolation**: The `vmrest` interface allows controlling local VMs via HTTP, granting high privileges. **Absolutely ensure** that the host's API listening port (default 55555) is not exposed to the public internet and is only accessible within a trusted LAN.
- **Disclaimer**: Bulk power operations (like `Power Off`) may result in the loss of unsaved data within the VMs. Please carefully verify the VM status before operating.
